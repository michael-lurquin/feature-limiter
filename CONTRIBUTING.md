## Development setup

```bash
git clone https://github.com/michael-lurquin/feature-limiter.git
cd feature-limiter
composer install
```

## Running tests

```bash
composer test
```

## Coding style

- Keep code consistent with the existing style
- Prefer small, focused PRs
- Add/adjust tests for any behavior change

## Pull requests

1.	Fork the repo
2.	Create a feature branch (feat/..., fix/...)
3.	Add tests
4.	Open a PR with a clear description

## Reporting issues

Please include:

- Laravel version
- PHP version
- Minimal reproduction steps
- Expected vs actual behavior
